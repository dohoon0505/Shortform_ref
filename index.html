템플릿 저장공간

기준 영상을 정하고, 항목별 프롬프트를 복사해 분석하세요. 각 템플릿은 후킹·구성·카피·편집·재현 5개 프롬프트로 이루어집니다.

사용법

기준 영상 하나를 정하고, 저장된 프롬프트로 그 영상을 해부한 뒤, 그 공식을 내 콘텐츠로 옮기는 흐름입니다.

핵심. 각 프롬프트는 그대로 복사해 ChatGPT·Claude 같은 AI에 붙여넣는 완성형입니다. 대괄호 [영상 링크]·[내 주제] 부분만 내 상황으로 바꾸면 됩니다.
  1. 기준 영상 정하기분석하고 싶은 잘된 숏폼(내 영상이든 벤치마크든)을 하나 고릅니다. 링크와 자막·스크립트를 준비하세요.
  2. 맞는 아키타입 템플릿 열기후킹 폭발형·스토리텔링형·정보형·UGC 광고형·밈 트렌드형 중 영상 성격에 맞는 템플릿을 저장공간에서 엽니다.
  3. 항목 프롬프트 복사 → AI에 붙여넣기후킹·구성·카피·편집 중 알고 싶은 항목의 복사 버튼을 누르고, AI에 붙여넣은 뒤 영상 링크·스크립트를 채워 실행합니다.
  4. ‘내 주제로 재현’으로 기획안 뽑기마지막 재현 프롬프트에 [내 주제]를 넣으면, 분석한 성공 공식이 바로 내 콘텐츠 기획안으로 바뀝니다.

팁

  • 순서대로 쓰면 좋아요. 후킹 → 구성 → 카피 → 편집 순으로 분석하면 맥락이 쌓여 재현 프롬프트의 결과가 좋아집니다.
  • 자막·스크립트를 최대한 채우세요. AI가 영상을 직접 보지 못하므로, 자막이 풍부할수록 분석 정확도가 올라갑니다.

템플릿 추가하기

저장공간은 templates.json 파일 하나로 관리됩니다. 이 파일에 객체를 추가하면 화면에 자동으로 나타납니다.

단일 진실의 출처. 화면은 templates.json을 읽어 렌더링합니다. HTML·JS를 건드릴 필요 없이, JSON에 템플릿 하나를 더하면 끝입니다.

1. templates.json 구조

templates 배열에 아래 형태의 객체를 추가하세요. prompts는 hook · structure · script · edit · remix 순서를 권장합니다.

{
  "id": "tpl-my-template",
  "title": "내 템플릿 이름",
  "archetype": "후킹 폭발형",
  "summary": "한 줄 설명",
  "reference": {
    "title": "기준 영상 성격",
    "platform": "YouTube Shorts",
    "creator": "(기준 영상 직접 지정)",
    "length": "0:40",
    "url": "",
    "why": "왜 이 영상을 기준으로 삼는지"
  },
  "bestFor": ["이럴 때 좋아요 …"],
  "pros": ["장점 …"],
  "cons": ["주의점 …"],
  "prompts": [
    { "id": "hook", "label": "후킹 분석", "desc": "…", "text": "그대로 복사해 쓰는 프롬프트 …" }
  ]
}

2. 반영 확인

  • archetype이 새 값이면 저장공간 상단과 사이드바에 필터 칩이 자동 생성됩니다.
  • reference.url을 채우면 카드에 영상이 임베드됩니다 — .mp4는 플레이어(<video>), 그 외 URL은 <iframe>. 영상 파일을 저장소에 커밋하지 마세요. 비워 두면 “기준 영상 직접 지정”으로 표시됩니다.
  • 기준 영상의 장면 분석은 힉스필드 MCP로 수행합니다 — 절차는 docs/video-analysis.md 참고.
  • 저장 후 브라우저를 새로고침하면 바로 나타납니다. CHANGELOG에 변경을 남겨 두면 이력이 누적됩니다.
