# 숏폼 연구소 · 템플릿 저장공간

> 잘된 숏폼 하나를, **분석 프롬프트**로 저장하다.

**숏폼 연구소**는 기준이 될 숏폼 영상(YouTube Shorts · Instagram Reels · TikTok) 하나를 정하고, 그 영상을 **후킹 · 구성 · 카피 · 편집** 항목별로 뜯어보는 **분석 프롬프트**를 저장해 두는 아카이브입니다. 필요할 때 프롬프트를 **복사**해 ChatGPT · Claude 같은 AI에 붙여넣고, **내 주제로 재현**하는 기획안까지 한 흐름으로 이어집니다.

- **현재 버전**: v1.7.0 (2026.07.11)
- **형태**: 바닐라 HTML/CSS/JS 단일 페이지 앱. 빌드 도구·프레임워크 없음.
- **단일 진실의 출처**: [`templates.json`](templates.json) — 화면은 이 파일을 읽어 렌더링합니다.
- **기준 영상**: 저장소에 파일로 두지 않고 `reference.url`로 **임베드**합니다(`.mp4` → `<video>`, 그 외 → `<iframe>`).
- **영상 분석**: 항상 **힉스필드(Higgsfield) MCP**로 수행 — 절차는 [docs/video-analysis.md](docs/video-analysis.md).

## 빠르게 실행

`templates.json`을 `fetch`로 읽기 때문에 `index.html`을 파일로 바로 열면(`file://`) 브라우저 보안 정책에 막힙니다. 간이 서버로 여세요.

```bash
npm run serve      # → http://127.0.0.1:8178  (Node만 있으면 됨, 의존성 0)
# 또는
npx serve          # 아무 정적 서버나 가능
python -m http.server 8178
```

## 템플릿이 담는 것

각 템플릿(=하나의 분석 세트)은 다음으로 구성됩니다.

| 항목 | 내용 |
| --- | --- |
| **기준 영상** (`reference`) | 플랫폼 · 길이 · 어떤 성격의 영상을 기준으로 삼는지 · 링크(비우면 “직접 지정”) |
| **사용성** (`bestFor`) | 이 템플릿을 언제 쓰면 좋은지 |
| **장단점** (`pros` / `cons`) | 강점과 주의점 |
| **분석 프롬프트** (`prompts`) | 항목별 완성형 프롬프트 5종 — 그대로 복사해 AI에 붙여넣기 |

프롬프트 5종은 순서대로 이어 쓰도록 설계되었습니다.

1. **후킹 분석** (`hook`) — 첫 3초가 이탈을 막는 원리
2. **구성·전개 분석** (`structure`) — 끝까지 보게 만드는 리텐션 구조
3. **카피·스크립트 분석** (`script`) — 말·자막의 문장 전략
4. **편집·비주얼 리듬 분석** (`edit`) — 컷·자막·사운드의 리듬
5. **내 주제로 재현하기** (`remix`) — 성공 공식을 내 콘텐츠 기획안으로

### 시드 템플릿 5종

후킹 폭발형 · 스토리텔링형 · 정보/튜토리얼형 · 리뷰/UGC 광고형 · 밈/트렌드·챌린지형.

## 파일 구조

```
숏폼 연구소/
├── index.html            ← 앱 셸 (사이드바 · 뷰 · 아이콘 스프라이트)
├── templates.json        ★ 단일 진실의 출처 (템플릿 데이터)
├── assets/
│   ├── css/main.css      ← 토큰(라이트/다크) + 전체 스타일
│   └── js/main.js        ← 라우터 · 렌더링 · 복사 · 필터/검색 · 테마
├── scripts/
│   ├── serve.mjs         ← 로컬 개발 서버 (의존성 0)
│   └── validate.mjs      ← templates.json 무결성 검증
├── docs/
│   └── video-analysis.md ★ 영상 분석 방법 (힉스필드 MCP · 분석 작업 전 필독)
├── AGENTS.md             ← AI 에이전트 진입점
├── CLAUDE.md             ← 기여(템플릿 추가) 워크플로
└── CHANGELOG.md          ← 변경 이력(누적)
```

## 템플릿 추가

[`templates.json`](templates.json)의 `templates` 배열에 객체 하나를 추가하면 끝입니다. 자세한 스키마와 절차는 [CLAUDE.md](CLAUDE.md) 또는 앱 안의 **템플릿 추가하기** 페이지를 보세요.

```bash
npm run validate   # 추가 후 무결성 검증
```
